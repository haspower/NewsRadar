## 1. 项目概述

### 1.1 项目简介

**TrendRadar（趋势雷达）** 是一款轻量级、可定制的热点新闻聚合与分析工具。它通过定时爬取多个主流热榜平台和 RSS 订阅源，结合 AI 大模型进行深度分析，将用户关心的热点内容精准推送到指定的通知渠道。

### 1.2 核心价值

- **告别无效刷屏**：通过关键词匹配或 AI 智能筛选，只推送用户真正关心的新闻
- **多平台聚合**：一次性获取今日头条、微博、知乎、抖音等 10+ 平台的热榜数据
- **AI 深度分析**：调用大模型对热点进行情感分析、趋势研判、策略建议
- **灵活调度**：支持 cron 定时或 GitHub Actions 周期性运行，最快 30 秒即可完成部署
- **多渠道推送**：支持飞书、钉钉、企业微信、Telegram、邮件等 9 种通知方式

### 1.3 支持的热榜平台

| 平台 ID | 平台名称 |
|---------|----------|
| toutiao | 今日头条 |
| baidu | 百度热搜 |

| bilibili-hot-search | Bilibili 热搜 |

## 2. 核心功能

### 2.1 数据采集

#### 热榜爬取
- 支持 10+ 主流热榜平台
- 可配置请求间隔，避免触发反爬
- 支持代理配置（本地环境）
- 失败平台自动记录

#### RSS 订阅
- 支持自定义 RSS 源
- 内置 Hacker News、阮一峰博客、雅虎财经等示例
- 文章新鲜度过滤（可配置保留天数）
- 每个源可独立配置过滤策略

### 2.2 筛选策略

#### 关键词匹配（默认）
- 基于 `config/frequency_words.txt` 配置的关注词库
- 支持词组匹配和全局过滤词
- 可自定义多个词库文件

#### AI 智能筛选
- 基于用户兴趣描述自动分类新闻
- 支持置信度阈值调节
- 增量更新标签（避免重复 token 消耗）
- 批量处理（每批 200 条标题）

### 2.3 报告模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **daily** | 当日所有新闻汇总 | 晚间复盘、全天总结 |
| **current** | 当前在榜新闻 | 实时热点监控 |
| **incremental** | 仅新增新闻 | 避免重复推送 |

### 2.4 展示模式

| 模式 | 说明 |
|------|------|
| **keyword** | 按关键词/标签分组显示 |
| **platform** | 按平台/来源分组显示 |

### 2.5 调度系统

基于 `timeline.yaml` 的时间线调度系统，支持：

- **预设模板**：always_on、morning_evening、office_hours、night_owl
- **自定义时间段**：自由定义每天的时间段和行为
- **周计划差异**：工作日和周末可使用不同策略
- **一次性控制**：每个时间段内只分析/推送一次，避免重复

#### 调度行为三要素

```
采集 (collect) → 分析 (analyze) → 推送 (push)
```

每个时间段可独立控制这三个行为的开关。

## 4. 模块设计

### 4.1 核心模块 (trendradar/core/)

#### 4.1.1 配置加载 (config.py)

负责加载和解析 `config.yaml`，提供：
- YAML 配置读取
- 环境变量覆盖支持
- 多账号配置解析
- 配置验证

#### 4.1.2 频率统计 (frequency.py)

关键词匹配和频率统计算法：
- 词组匹配（多词组合）
- 全局过滤词
- 频率统计
- 排名权重计算

#### 4.1.3 调度器 (scheduler.py)

时间线调度核心逻辑：
- 预设模板加载
- 自定义时间段解析
- 跨日时间段处理（如 22:00-07:00）
- 冲突检测策略（error_on_overlap / last_wins）
- once 执行去重（analyze / push 独立维度）

### 4.2 爬虫模块 (trendradar/crawler/)

#### 4.2.1 热榜爬取 (fetcher.py)

- 多平台并发/串行爬取
- 请求间隔控制
- 代理支持
- 错误处理和重试

#### 4.2.2 RSS 爬取 (rss/fetcher.py)

- feedparser 解析
- 多源并发请求
- 文章新鲜度过滤
- 新增检测

### 4.3 AI 模块 (trendradar/ai/)

#### 4.3.1 AI 分析器 (analyzer.py)

核心 AI 分析功能：
- 五大分析板块：
  1. 核心趋势与舆情态势
  2. 舆论风向与争议分析
  3. 异动信号与弱信号检测
  4. RSS 深度洞察
  5. 研判与策略建议
- JSON 响应解析与修复
- 独立展示区概括

#### 4.3.2 AI 筛选 (filter.py)

智能新闻筛选：
- 兴趣标签提取
- 批量分类（200 条/批）
- 增量更新（change_ratio 阈值触发全量重分类）
- 置信度过滤

#### 4.3.3 AI 翻译 (translator.py)

多语言翻译：
- 支持热榜、RSS、独立展示区翻译
- 可配置翻译范围
- 目标语言自定义

### 4.4 存储模块 (trendradar/storage/)

#### 4.4.1 存储管理器 (manager.py)

统一存储接口：
- 后端自动选择（auto/local/remote）
- 数据格式管理（SQLite/TXT/HTML）
- 数据保留策略
- 过期数据清理

#### 4.4.2 本地存储 (local.py)

- SQLite 主存储
- TXT 快照生成
- HTML 报告生成

#### 4.4.3 远程存储 (remote.py)

- S3 兼容协议
- 支持 Cloudflare R2、阿里云 OSS、腾讯云 COS、AWS S3、MinIO
- 自动同步

### 4.5 通知模块 (trendradar/notification/)

#### 4.5.1 通知分发器 (dispatcher.py)

- 多渠道分发
- 内容分块（适配各渠道限制）
- 格式转换（Markdown → 各渠道格式）

#### 4.5.2 发送器 (senders.py)

各渠道具体实现：
- Webhook 发送（飞书、钉钉、企业微信、Slack）
- Bot API（Telegram）
- SMTP（邮件）
- HTTP POST（ntfy、Bark、通用 Webhook）

## 5. 部署方式

### 5.1 GitHub Actions 部署（推荐新手）

**优势**：
- 免费云服务器
- 无需自建服务器
- 一键 fork 即可运行

**部署步骤**：
1. Fork 项目到个人仓库
2. 在 GitHub Secrets 中配置敏感信息（API Key、Webhook 等）
3. 修改 `.github/workflows/crawler.yml` 中的 cron 表达式
4. Commit 后自动触发运行

**定时配置**：
```yaml
# .github/workflows/crawler.yml
schedule:
  - cron: "33 * * * *"  # 每小时第 33 分钟运行
```

### 5.2 Docker 部署

**优势**：
- 本地/私有服务器部署
- 更灵活的定时控制
- 数据完全自主

**部署步骤**：
```bash
# 1. 克隆项目
git clone https://github.com/sansan0/TrendRadar.git
cd TrendRadar

# 2. 配置环境变量
cp docker/.env.example docker/.env
# 编辑 docker/.env 填写配置

# 3. 启动容器
docker compose up -d
```

**定时配置**：
```bash
# docker/.env
CRON_SCHEDULE="*/30 * * * *"  # 每 30 分钟运行
```

### 5.3 本地开发部署

```bash
# 1. 创建虚拟环境
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# 2. 安装依赖
pip install -r requirements.txt

# 3. 运行程序
python -m trendradar

# 4. 运行 MCP Server
python -m mcp_server
```

```yaml
# AI 模型
ai:
  model: "deepseek/deepseek-chat"
  api_key: "${AI_API_KEY}"

# AI 分析
ai_analysis:
  enabled: true
  language: "Chinese"
  max_news_for_analysis: 150

# 存储
storage:
  backend: "auto"
  formats:
    sqlite: true
    html: true
```

### 6.2 时间线配置 (config/timeline.yaml)

定义调度行为：

```yaml
presets:
  morning_evening:
    name: "早晚汇总"
    default:
      collect: true
      analyze: true
      push: true
      report_mode: "current"
    periods:
      evening_summary:
        name: "晚间汇总"
        start: "20:00"
        end: "22:00"
        analyze: true
        ai_mode: "daily"
        report_mode: "daily"
        once:
          analyze: true
          push: true
    day_plans:
      all_day:
        periods: ["evening_summary"]
    week_map:
      1: "all_day"  # 周一
      2: "all_day"
      # ...
```

## 7. MCP Server

### 7.1 概述

TrendRadar 提供基于 **FastMCP 2.0** 的 MCP（Model Context Protocol）服务器，可将新闻数据和分析能力开放给 AI 助手（如 Claude Desktop、Cherry Studio 等）。

### 7.2 启动方式

```bash
# stdio 模式（本地客户端）
python -m mcp_server --transport stdio

# HTTP 模式（生产环境）
python -m mcp_server --transport http --port 3333
```

### 7.3 工具列表

| 工具名称 | 功能描述 |
|----------|----------|
| `resolve_date_range` | 解析自然语言日期为标准格式 |
| `get_latest_news` | 获取最新新闻数据 |
| `get_trending_topics` | 获取热点话题统计 |
| `search_news` | 统一新闻搜索（关键词/模糊/实体） |
| `find_related_news` | 查找相关新闻 |
| `get_latest_rss` | 获取最新 RSS 订阅数据 |
| `search_rss` | 搜索 RSS 数据 |
| `analyze_topic_trend` | 话题趋势分析 |
| `analyze_sentiment` | 情感倾向分析 |
| `aggregate_news` | 跨平台新闻聚合去重 |
| `compare_periods` | 时期对比分析 |
| `generate_summary_report` | 每日/每周摘要生成 |
| `read_article` | 读取文章内容（Markdown 格式） |
| `send_notification` | 发送通知到各渠道 |
| `get_current_config` | 获取当前配置 |
| `trigger_crawl` | 手动触发爬取 |
| `sync_from_remote` | 从远程存储拉取数据 |

### 7.4 使用示例

在 AI 对话中：

```
用户：帮我分析一下今天知乎和微博上关于 AI 的热点话题

AI（调用 MCP 工具）：
1. resolve_date_range("今天") → 获取精确日期范围
2. get_trending_topics(mode="current", extract_mode="keywords") → 获取热点
3. search_news(query="AI", platforms=["zhihu", "weibo"]) → 搜索 AI 相关新闻
4. analyze_sentiment(topic="AI", platforms=["zhihu", "weibo"]) → 情感分析
```

### 8.2 多账号配置

使用分号分隔：

```yaml
notification:
  channels:
    telegram:
      bot_token: "token1;token2;token3"
      chat_id: "chat1;chat2;chat3"
```

最多支持 3 个账号/渠道。

### 8.3 格式适配

各渠道 Markdown 特性：

| 渠道 | 支持特性 |
|------|----------|

## 9. AI 功能

### 9.1 AI 分析

**配置**：
```yaml
ai_analysis:
  enabled: true
  language: "Chinese"
  max_news_for_analysis: 150
  include_rss: false
  include_standalone: true
  include_rank_timeline: true
  mode: "follow_report"  # daily/current/incremental
```

**输出内容**：
1. 核心趋势与舆情态势
2. 舆论风向与争议分析
3. 异动信号与弱信号检测
4. RSS 深度洞察（可选）
5. 研判与策略建议
6. 独立展示区概括（可选）

### 9.2 AI 筛选

**配置**：
```yaml
filter:
  method: "ai"

ai_filter:
  batch_size: 200
  batch_interval: 2
  min_score: 0.7
  reclassify_threshold: 0.6
```

**工作原理**：
1. 读取 `config/ai_interests.txt` 中的兴趣描述
2. 每批 200 条标题发送给 AI 进行分类
3. AI 返回每个标题的标签和匹配分数
4. 过滤低于 `min_score` 的新闻
5. 当新闻变化比例超过 `reclassify_threshold` 时全量重分类

### 9.3 AI 翻译

**配置**：
```yaml
ai_translation:
  enabled: true
  language: "中文"
  scope:
    hotlist: false
    rss: true
    standalone: true
```

**翻译范围**：
- 热榜区域（可选）
- RSS 区域（可选）
- 独立展示区（可选）

### 9.4 支持的 AI 模型

基于 LiteLLM，支持 100+ 模型：

```yaml
ai:
  model: "deepseek/deepseek-chat"  # DeepSeek
  # model: "openai/gpt-4o"         # OpenAI
  # model: "gemini/gemini-2.5-flash"  # Google
  # model: "anthropic/claude-3-5-sonnet"  # Anthropic
  # model: "ollama/llama3"         # 本地 Ollama

  api_key: "${AI_API_KEY}"
  api_base: ""  # 自定义 API 端点（可选）
  timeout: 120
  temperature: 1.0
  max_tokens: 5000
```

## 10. 数据存储

### 10.1 存储后端

| 后端 | 说明 | 适用场景 |
|------|------|----------|
| **auto** | 自动选择（GitHub Actions+远程配置→remote，否则→local） | 大多数用户 |
| **local** | 本地 SQLite + TXT/HTML 文件 | 本地/Docker 部署 |
| **remote** | 远程 S3 兼容存储 | GitHub Actions 部署 |

### 10.2 数据格式

```yaml
storage:
  formats:
    sqlite: true   # 主存储（必须）
    txt: false     # TXT 快照
    html: true     # HTML 报告（邮件推送必备）
```

### 10.3 数据保留

```yaml
storage:
  retention_days: 0  # 0=永久保留
```

### 10.4 远程存储配置

支持 S3 兼容协议：

```yaml
storage:
  remote:
    endpoint_url: "https://<account_id>.r2.cloudflarestorage.com"
    bucket_name: "trendradar"
    access_key_id: "${S3_ACCESS_KEY_ID}"
    secret_access_key: "${S3_SECRET_ACCESS_KEY}"
    region: "auto"
```

### 10.5 数据拉取配置

MCP Server 场景：

```yaml
storage:
  pull:
    enabled: true
    days: 7  # 拉取最近 7 天数据
```

爬虫程序将数据存到远程云存储，MCP Server 启动时自动拉取到本地进行分析查询。
